## 1. 구현된 기능 목록

### ✅ 데이터 구조 (SO)
| SO 클래스 | 상태 |
|---|---|
| `IngredientSO` (카테고리 enum 포함) | 완료 |
| `RecipeSO` (일반/전용 메뉴, normalCounterpart 필드 포함) | 완료 |
| `GhostSO` | 완료 |
| `ArtifactSO` | 완료 |
| `DialogueSO` | 껍데기만 생성

### ✅ 조리 시스템
- 재료 투입 → 조리 시작 → 타이머 → 수령 전체 흐름 작동
- 슬롯 상태 머신 (`Empty / Filling / Cooking / Ready / Spoiled`) 구현
- 멀티슬롯 독립 동작, 활성 슬롯 전환
- 조리 중도 취소 가능
- 일정 시간 방치 시 음식 상함(`Spoiled`) 전환

### ✅ 레시피 검증 (`RecipeValidator.cs`)
- 일반 메뉴: 순서 일치 검증 
- 전용 메뉴: 단골 본인 주문 시 , 타 손님 주문 시  분기 검증
- 오답 시 Soft Fail (망한 음식 처리, 영업 계속)

### ✅ 재료 패널 UI
- 카테고리 탭 분리
- 탭 전환 시 해당 카테고리 재료 버튼 동적 표시

### ✅ 드래그 서빙
- 완성된 음식을 손님에게 드래그하여 전달
- 잘못된 손님에게 드래그 시 슬롯으로 복귀

### ✅ 손님 시스템
- 손님 스폰 루프 (`GuestSpawner`) — 일정 간격으로 등장
- 손님 진입/퇴장 이동 연출, 방향에 따른 스프라이트 Flip
- 주문 팝업 (`OrderPopup`) — 메뉴 아이콘 표시
- 인내심 게이지 (`PatienceGauge`) — 소진 시 손님 이탈

### ✅ 영업 세션
- 세션 타이머 (`SessionManager`) 작동
- 세션 종료 시 영업 마감 UI 표시
- 마감 전 남은 손님 처리 후 닫기

## 2. 임시 처리된 부분

| 위치 | 내용 | 비고 |
|---|---|---|
| `SessionManager.cs` `_sessionDuration` | 현재 **120**로 설정됨 | 정식 값은 120~180초 |
| `CookingSlotUI.cs` `_testCookTime` | 조리 시간 **5f 하드코딩** | 추후 `RecipeSO.cookTime` 연동 필요 |
| `IngredientPanelUI.cs` | 지금은 UI로 만들었지만 추후 인게임 오브젝트로 수정 
| `GuestSpawner.cs` 스폰 루프 | `while(true)` 무한 루프 | 영업 마감 시스템 연동 전까지 임시 |
| `OrderPopup.cs` 게이지 아이콘 | SpriteRenderer 미연결 주석 있음 | 아트 에셋 추가 후 연결 필요 |
| 레시피 노출 | **임시로 레시피 힌트 노출 중** | 정식 빌드에서는 주문 메뉴만 표시 예정 |
| 방문하는 손님, 레시피 | 모든 타입의 손님이 방문하고, 모든 레시피가 해금되어 있음 | 추후에는 스토리 진행 흐름에 따라 해금 (손님, 레시피 프리팹은 1/2만 만들어서 테스트함)

## 4. 플레이에 필요한 정보

### 기본 조작
1. **슬롯 탭** → 해당 슬롯 활성화 (테두리 강조)
2. **재료 패널 탭** → 활성 슬롯에 재료 순서대로 투입
3. **조리 시작 버튼 탭** → 타이머 시작
4. **타이머 종료** → 슬롯에서 완성 음식 드래그 → 손님에게 전달

### 주의사항
- **레시피 순서가 중요합니다.** 재료 종류가 맞아도 순서가 틀리면 망한 음식이 됩니다.
- 전용 메뉴는 주문한 손님이 단골 본인인지 아닌지에 따라 마지막 재료가 달라집니다.
- 현재 빌드에서 **레시피 힌트가 임시로 노출** 중입니다.
- 골드/EXP 적립 기능은 아직 없습니다.

### 현재 플레이 가능한 범위
> 손님 등장 → 주문 → 재료 투입 및 조리 → 완성 음식 드래그 서빙 → 손님 만족/이탈 → 세션 타이머 종료 및 마감 UI -> 로비 이동
